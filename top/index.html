1 安装 Node.js 环境

Claude Code 需要 Node.js 环境才能运行。请按照以下步骤安装 Node.js：

下载 Node.js

打开浏览器访问 https://nodejs.org/，点击 "LTS" 版本进行下载。

安装 Node.js

下载完成后双击 .msi 文件，按照安装向导完成安装。

验证安装

打开 PowerShell，运行以下命令验证安装是否成功：

node --version
npm --version
✅ 验证成功：如果显示版本号（如 v22.x.x），说明 Node.js 安装成功！
注意事项：
  • 建议使用 PowerShell 而不是 CMD
  • 如果遇到权限问题，尝试以管理员身份运行
  • 某些杀毒软件可能会误报，请添加信任

2 安装 Git Bash

Windows 环境下需要使用 Git Bash 安装 Claude Code。安装完成后，环境变量设置和使用仍在 PowerShell 或 CMD 中。

下载 Git

访问 https://git-scm.com/downloads/win 下载 Git for Windows。

安装 Git

下载完成后运行安装程序，按照默认设置完成安装即可。

验证安装

git --version

3 安装 Claude Code

打开 PowerShell，运行以下命令安装 Claude Code：

npm install -g @anthropic-ai/claude-code

如果安装过程中提示需要将 ~/.local/bin 加入 PATH，请运行：

[Environment]::SetEnvironmentVariable('Path', ([Environment]::GetEnvironmentVariable('Path','User') + ";$HOME\.local\bin"), 'User')

验证安装

claude --version
✅ 验证成功：如果显示版本号，说明 Claude Code 安装成功！

5 设置环境变量

为了让 Claude Code 连接到中转服务，需要将 ANTHROPIC_BASE_URL 设置为：

https://api.augmentproxy.top

方式一：一键设置推荐

运行以下命令，按提示输入上述地址和你的 API Key 即可自动完成配置：

iex (irm 'https://raw.githubusercontent.com/QuantumNous/new-api-docs/refs/heads/main/helper/claude-cli-setup.ps1')
方式二：手动设置（点击展开）

临时设置（仅当前会话有效）

$env:ANTHROPIC_BASE_URL = "https://api.augmentproxy.top"
$env:ANTHROPIC_AUTH_TOKEN = "your_api_key"

永久设置（推荐）

[System.Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL", "https://api.augmentproxy.top", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN", "your_api_key", [System.EnvironmentVariableTarget]::User)
获取 API Key：请参考上方第 4 步获取你的 API Key。
提示：设置永久环境变量后，需要重新打开 PowerShell 窗口才能生效。
✅ 验证方法：重新打开 PowerShell，运行 echo $env:ANTHROPIC_BASE_URL，如果输出 https://api.augmentproxy.top，说明环境变量设置成功！

API Key 就像是你的"通行证"或"门禁卡"。当你使用 Claude Code 等工具时，需要用 API Key 来证明你的身份，这样服务器才知道是你在使用服务，并从你的账户扣费。每个人的 API Key 都是唯一的，以 sk- 开头。

中转服务只做请求转发，不存储你的对话数据和代码内容。你的请求通过中转服务器转发到 AI 模型提供商（如 Anthropic），响应结果直接返回给你。整个过程使用 HTTPS 加密传输，确保数据安全。

  3. 验证设置：echo $env:ANTHROPIC_AUTH_TOKEN
  4. 如仍不生效，尝试注销并重新登录 Windows

4 配置 OpenClaw

打开你的 OpenClaw 配置文件 ~/.openclaw/openclaw.json，将其中的 models 和 agents 字段替换为以下内容：

⚠️ 注意：只需替换 models 和 agents 两个字段，配置文件中的其他字段（如 channels、gateway、plugins 等）保持不变。

替换以下字段内容

"models": {
  "providers": {
    "augmentproxytop": {
      "baseUrl": "https://api.augmentproxy.top",
      "apiKey": "<YOUR_API_KEY>",
      "api": "anthropic-messages",
      "headers": {
        "User-Agent": "curl/8.0"
      },
      "authHeader": true,
      "models": [
        {
          "id": "claude-opus-4-6",
          "name": "Claude Opus 4.6",
          "reasoning": true,
          "input": ["text", "image"],
          "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
          "contextWindow": 200000,
          "maxTokens": 128000
        }
      ]
    },
    "openaixyz": {
      "baseUrl": "https://api.augmentproxy.top/v1",
      "apiKey": "<YOUR_API_KEY>",
      "api": "openai-completions",
      "headers": {
        "User-Agent": "curl/8.0"
      },
      "authHeader": true,
      "models": [
        {
          "id": "gpt-5.3-codex",
          "name": "GPT-5.3 Codex",
          "reasoning": true,
          "input": ["text", "image"],
          "cost": { "input": 1.75, "output": 14, "cacheRead": 0.175, "cacheWrite": 1.75 },
          "contextWindow": 400000,
          "maxTokens": 128000
        }
      ]
    }
  }
},
"agents": {
  "defaults": {
    "model": {
      "primary": "augmentproxytop/claude-opus-4-6",
      "fallbacks": ["openaixyz/gpt-5.3-codex"]
    },
    "models": {
      "augmentproxytop/claude-opus-4-6": {},
      "openaixyz/gpt-5.3-codex": {}
    },
    "workspace": "<your_workspace_dir>",
    "compaction": {
      "mode": "safeguard",
      "reserveTokensFloor": 20000,
      "maxHistoryShare": 0.7,
      "memoryFlush": { "enabled": true, "softThresholdTokens": 10000 }
    },
    "maxConcurrent": 4,
    "subagents": { "maxConcurrent": 8 }
  },
  "list": [{ "id": "main" }]
},
占位符说明：
  • <YOUR_API_KEY>：替换为你在 api.augmentproxy.top 网站上获取的 API 令牌（两个 provider 需分别使用各自的 Key）
  • <your_workspace_dir>：保持你原有配置中的 workspace 路径不变（如 ~/clawd）
💡 配置亮点：
  • claude-opus-4-6：主力模型，200K 上下文窗口，最大输出 128K tokens
  • gpt-5.3-codex：备用模型，400K 上下文窗口，主模型不可用时自动切换
  • fallbacks：自动故障转移，主模型挂了无缝切到备用模型
  • compaction：上下文压缩，自动管理长对话，避免 502 错误
  • reserveTokensFloor：保留 20K tokens 给模型回复，防止上下文撑满

替换以下字段内容

"models": {
  "providers": {
    "augmentproxytop": {
      "baseUrl": "https://api.augmentproxy.top",
      "apiKey": "<YOUR_API_KEY>",
      "api": "anthropic-messages",
      "headers": {
        "User-Agent": "curl/8.0"
      },
      "authHeader": true,
      "models": [
        {
          "id": "claude-opus-4-6",
          "name": "Claude Opus 4.6",
          "reasoning": true,
          "input": ["text", "image"],
          "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
          "contextWindow": 200000,
          "maxTokens": 128000
        }
      ]
    },
    "openaixyz": {
      "baseUrl": "https://api.augmentproxy.top/v1",
      "apiKey": "<YOUR_API_KEY>",
      "api": "openai-completions",
      "headers": {
        "User-Agent": "curl/8.0"
      },
      "authHeader": true,
      "models": [
        {
          "id": "gpt-5.3-codex",
          "name": "GPT-5.3 Codex",
          "reasoning": true,
          "input": ["text", "image"],
          "cost": { "input": 1.75, "output": 14, "cacheRead": 0.175, "cacheWrite": 1.75 },
          "contextWindow": 400000,
          "maxTokens": 128000
        }
      ]
    }
  }
},
"agents": {
  "defaults": {
    "model": {
      "primary": "augmentproxytop/claude-opus-4-6",
      "fallbacks": ["openaixyz/gpt-5.3-codex"]
    },
    "models": {
      "augmentproxytop/claude-opus-4-6": {},
      "openaixyz/gpt-5.3-codex": {}
    },
    "workspace": "<your_workspace_dir>",
    "compaction": {
      "mode": "safeguard",
      "reserveTokensFloor": 20000,
      "maxHistoryShare": 0.7,
      "memoryFlush": { "enabled": true, "softThresholdTokens": 10000 }
    },
    "maxConcurrent": 4,
    "subagents": { "maxConcurrent": 8 }
  },
  "list": [{ "id": "main" }]
},
占位符说明：
  • <YOUR_API_KEY>：替换为你在 api.augmentproxy.top 网站上获取的 API 令牌（两个 provider 需分别使用各自的 Key）
  • <your_workspace_dir>：保持你原有配置中的 workspace 路径不变（如 ~/clawd）
💡 配置亮点：
  • claude-opus-4-6：主力模型，200K 上下文窗口，最大输出 128K tokens
  • gpt-5.3-codex：备用模型，400K 上下文窗口，主模型不可用时自动切换
  • fallbacks：自动故障转移，主模型挂了无缝切到备用模型
  • compaction：上下文压缩，自动管理长对话，避免 502 错误
  • reserveTokensFloor：保留 20K tokens 给模型回复，防止上下文撑满

Fedora/RHEL
sudo dnf install nodejs
Arch Linux
sudo pacman -S nodejs npm

npm i -g opencode-ai

bun add -g opencode-ai

paru -S opencode

在 VS Code / Cursor 等支持 Claude Code 插件的 IDE 中使用 Claude Code

VS Code 和 Cursor 的配置方式完全一致，按以下步骤配置即可通过 API 中转使用 Claude Code 插件。

🔌 Claude Code 官方插件
⚙️ CC Switch 一键配置
🚀 无需官方账号登录
💡 VS Code / Cursor 通用

1 安装 Claude Code 插件

在 VS Code 或 Cursor 的扩展商店中搜索 Claude Code（由 Anthropic 官方发布），点击安装。

💡 提示：VS Code 和 Cursor 的配置方式完全一致，以下步骤通用。

2 安装 CC Switch 配置工具

CC Switch 是一个开源桌面应用，可以一键配置 Claude Code 的 API 中转地址和密钥。

前往 GitHub Releases 下载对应平台的安装包：

  • Windows：下载 .exe 或 .msi 安装包
  • macOS：下载 .dmg 安装包
  • Linux：下载 .AppImage 或 .deb
⚠️ 安装时遇到安全提示？ Windows：弹出 SmartScreen 提示时，点击「更多信息」→「仍要运行」。
macOS：提示"无法验证开发者"时，前往「系统设置 → 隐私与安全性」，找到 CC Switch 点击「仍要打开」。
Linux：AppImage 需要先赋予执行权限：chmod +x cc-switch-*.AppImage

3 配置 API 中转

打开 CC Switch，添加一个新的 Provider：

  • API Base URL 填写：https://api.augmentproxy.top
  • API Key 填写你在控制台获取的密钥
  • 在设置中开启「应用到 Claude Code 插件」选项

配置完成后，CC Switch 会自动将中转地址写入 Claude Code 的配置中。

4 跳过官方登录页面

使用 API 中转时，需要跳过 Claude 官方的登录验证。通常 CC Switch 配置后即可自动跳过，如果仍然弹出登录页面，请按顺序尝试以下方法：

方法一：修改 config.json

创建或编辑 ~/.claude/config.json，写入以下内容后重启插件：

{
    "primaryApiKey": "augmentproxytop"
}

方法二：禁用登录提示

在 VS Code / Cursor 设置中搜索 Claude Code，勾选 Disable Login Prompt 选项。

方法三：跳过初次安装引导

在 ~/.claude.json（与 .claude 文件夹同级）中添加以下字段：

{
    "hasCompletedOnboarding": true
}
✅ 验证成功：配置完成后，在 VS Code / Cursor 中打开 Claude Code 面板，如果能正常对话则说明配置成功。如果仍无法使用，请检查控制台钱包余额是否充足。
